# StudyGuard 智能自习室座位管理终端 副本

# 1\. 项目名称

**StudyGuard 智能自习室座位管理终端**

基于 **U1P \+ U2P 协同架构** 的自习室座位占用与环境监测系统。

---

## 2\. 项目背景

在图书馆、自习室等公共学习场景中，经常存在座位状态不透明、临时离开难以判断、长时间占座无人管理、环境舒适度无法及时反馈等问题。传统座位检测方案通常只判断“有人”或“无人”，功能比较单一，不能很好地区分正常使用、临时离开和超时占座等情况。

本项目设计一个基于 **U1P \+ U2P 双主控协同** 的智能座位管理终端。U1P 负责底层传感器采集与串口通信，U2P 负责触摸屏界面显示和上层状态判断。系统通过 **S7 人体感应、S5 NFC/RFID 刷卡识别、S8 温湿度检测** 等模块，实现座位状态判断、刷卡签到、临时离开提醒和环境异常提示。

相比原先使用 S4 摄像头的方案，本版本将座位识别方式改为 S7 人体感应，降低了图像识别和现场调试难度，更适合课程项目的稳定实现和答辩演示。

---

## 3\. 设计目标

本系统需要完成以下目标：

1. 使用 U1P 和 U2P 两块 U 板构成协同控制架构。

2. 使用 S7 人体感应模块判断座位前是否有人。

3. 使用 E5 HDMI 触摸屏显示座位状态、学生刷卡信息和环境数据。

4. 使用 S5 NFC/RFID 模块模拟学生刷卡预约、签到和签退。

5. 使用 S8 温湿度传感器采集自习室环境舒适度。

6. U1P 与 U2P 通过 UART 通信，实现底层数据采集与上层 UI 逻辑协同。

7. 实现空闲、预约、使用中、临时离开、疑似占座和环境异常等状态判断。

## 5\. 系统总体架构

系统采用 **“U1P 底层实时采集 \+ U2P 上层状态管理”** 的分层架构。

```Plain Text
S7 人体感应  ---- GPIO/I2C ----\
S5 NFC/RFID  ---- I2C --------- U1P ---- UART ---- U2P ---- HDMI/USB ---- E5 触摸屏
S8 温湿度    ---- I2C ---------/
```

各主控分工如下：

```Plain Text
U1P：
1. 初始化 GPIO/I2C/UART 等底层接口。
2. 读取 S7 人体感应信号。
3. 读取 S5 NFC/RFID 刷卡数据。
4. 读取 S8 温湿度数据。
5. 对传感器数据进行基础校验和格式化。
6. 通过 UART 将数据发送给 U2P。

U2P：
1. 运行 Ubuntu/Linux 系统。
2. 接收 U1P 上传的人体感应、刷卡和温湿度数据。
3. 根据刷卡状态、人体感应状态和计时器进行座位状态判断。
4. 运行图形界面程序，在 E5 触摸屏显示座位地图。
5. 提供座位详情、管理员清除、刷新等交互功能。
```

## 6\. 功能设计

### 6\.1 座位状态定义

系统将座位状态分为 6 类：

|状态|含义|显示颜色建议|
|---|---|---|
|EMPTY|空闲，无人且未预约|绿色|
|RESERVED|已刷卡预约，但尚未入座|蓝色|
|USING|已签到并检测到有人使用|深蓝色|
|AWAY|已签到用户短时间离开|黄色|
|OCCUPY|临时离开超时，疑似占座|红色|
|BAD\_ENV|当前温湿度不适合学习|白色或橙色提示|

其中，OCCUPY 不再表示摄像头识别到物品，而是表示 **已绑定座位的用户长时间离开未归**，系统将其判断为疑似占座或超时占用。

---

### 6\.2 刷卡签到功能

S5 NFC/RFID 用于模拟学生身份识别。

```Plain Text
学生刷卡
-> U1P 读取卡号
-> U1P 通过 UART 发送给 U2P
-> U2P 在界面上绑定座位
-> 座位状态变为 RESERVED 或 USING
```

刷卡逻辑：

```Plain Text
空闲座位刷卡       -> 预约或签到
已签到座位再次刷卡 -> 签退并释放座位
管理员卡刷卡       -> 清除异常占座状态
```

---

### 6\.3 人体感应检测功能

S7 人体感应模块用于判断座位前是否有人。

```Plain Text
S7 检测到人
-> human = 1

S7 未检测到人
-> human = 0
```

为了减少误判，系统可以加入简单防抖逻辑：

```Plain Text
human = 1 持续 3 秒以上
-> 判断为有人入座

human = 0 持续 5 秒以上
-> 判断为人员离开
```

这样可以避免由于人体感应短暂波动导致状态频繁切换。

---

### 6\.4 临时离开与疑似占座判断

系统通过刷卡状态、人体感应结果和计时器判断座位状态。

```Plain Text
未刷卡 + 无人
-> EMPTY

刷卡后 + 尚未检测到人
-> RESERVED

刷卡后 + S7 检测到人
-> USING

USING 状态下 + S7 短时间检测不到人
-> AWAY

AWAY 超过 5 分钟仍无人
-> OCCUPY

用户再次刷卡
-> 签退并释放座位
```

演示时可将 5 分钟缩短为 20 秒，方便现场展示。

---

### 6\.5 环境监测功能

S8 温湿度传感器由 U1P 读取，数据通过 UART 上传给 U2P。

环境判断规则建议：

```Plain Text
温度 < 18℃      -> 偏冷
温度 > 30℃      -> 偏热
湿度 > 75%RH    -> 潮湿
湿度 < 30%RH    -> 干燥
```

当环境异常时，E5 屏幕显示 BAD\_ENV 提示，并在座位地图旁显示当前温度和湿度。

---

## 7\. 通信协议设计

U1P 与 U2P 之间使用 UART 通信，采用一行一帧的文本协议，方便调试和演示。

### 7\.1 U1P 发送给 U2P

人体感应数据：

```Plain Text
HUMAN,seat=A01,value=1
HUMAN,seat=A01,value=0
```

刷卡数据：

```Plain Text
RFID,card=20260001,event=scan
```

温湿度数据：

```Plain Text
ENV,temp=26.5,humi=48.2
```

心跳数据：

```Plain Text
HB,u1p=ok
```

### 7\.2 U2P 发送给 U1P

状态确认：

```Plain Text
ACK,rfid=20260001,result=ok
```

请求环境数据：

```Plain Text
GET,env
```

系统复位：

```Plain Text
CMD,reset
```

## 9\. 状态机设计

单个座位的状态转换如下：

```Plain Text
EMPTY
  刷卡预约 -> RESERVED

RESERVED
  检测到人入座 -> USING
  超时未入座 -> EMPTY
  再次刷卡 -> EMPTY

USING
  人离开 -> AWAY
  再次刷卡 -> EMPTY

AWAY
  人返回 -> USING
  超时未归 -> OCCUPY
  再次刷卡 -> EMPTY

OCCUPY
  管理员清除 -> EMPTY
  用户刷卡确认 -> USING
```

状态判断重点不是单纯读取传感器，而是将 **刷卡身份、人体感应和时间规则** 结合起来，实现更接近真实自习室场景的状态管理。

## 10\. E5 触摸屏界面设计

界面分为三个区域：

```Plain Text
左侧：座位地图
  显示 2 到 4 个座位，每个座位用不同颜色表示状态。

右侧：座位详情
  显示座位编号、绑定卡号、当前状态、离开计时。

底部：环境信息
  显示温度、湿度和环境舒适度提示。
```

触摸操作可以设计为：

```Plain Text
点击座位       -> 查看座位详情
点击管理员清除 -> 清除 OCCUPY 状态
点击刷新       -> 重新读取传感器数据
```

## 11\. 使用场景

### 场景一：系统启动

```Plain Text
U1P 初始化 GPIO/I2C 和 UART。
U2P 启动触摸屏界面。
E5 显示 4 个绿色空闲座位。
```

### 场景二：学生刷卡预约

```Plain Text
学生将卡靠近 S5。
U1P 读取卡号并发送给 U2P。
U2P 将某个座位绑定给该学生。
E5 上该座位变为 RESERVED。
```

### 场景三：学生入座学习

```Plain Text
学生坐到对应座位前。
S7 检测到有人。
U1P 将 human=1 发送给 U2P。
座位状态由 RESERVED 变为 USING。
```

### 场景四：临时离开

```Plain Text
学生离开座位。
S7 检测不到人。
系统进入 AWAY 状态并开始倒计时。
倒计时内返回则恢复 USING。
```

### 场景五：疑似占座

```Plain Text
学生离开超过设定时间仍未返回。
系统将 AWAY 状态转为 OCCUPY。
E5 显示红色异常提示。
```

### 场景六：环境异常

```Plain Text
改变 S8 温湿度传感器周围环境。
U1P 读取温湿度并上传给 U2P。
U2P 判断温度或湿度异常。
E5 显示 BAD_ENV 提示。
```

## 13\. 创新点

相比普通的座位检测系统，本项目有以下特点：

1. 使用 U1P \+ U2P 双主控架构，体现 MCU 底层采集与 Linux UI 平台协同。

2. 采用 S7 人体感应、S5 刷卡识别和计时器融合判断，不是简单判断有人或无人。

3. 引入 NFC/RFID 身份识别，支持预约、签到、签退和管理员清除逻辑。

4. 使用 E5 触摸屏实现可视化座位地图，交互效果比数码管显示更完整。

5. 融合温湿度环境数据，形成座位管理与学习环境监测一体化系统。

6. 状态机逻辑完整，能够区分空闲、预约、使用中、临时离开和疑似占座。
